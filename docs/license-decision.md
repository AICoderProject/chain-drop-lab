# ライセンス選定および方針の検討 (License Decision)

本プロジェクトは、他社の商標や第三者の著作アセットを一切含まない完全オリジナルのオープンソース教材ゲームです。
公開にあたり、適切なオープンソースライセンスを適用するための比較と推奨候補を解説します。

---

## 1. 推奨されるライセンス候補：MIT License

本プロジェクトには **MITライセンス** の適用を推奨します。

### MITライセンスの特徴
- **極めて寛容 (Permissive)**:
  誰でも自由にコードをコピー、改変、配布、さらには商業利用（有料ゲームや他製品への組み込み）することができます。
- **著作権表示のみ**:
  コードを再利用する際に、原著作者の著作権表示（Copyright notice）を残すことだけが義務付けられます。
- **教育用に最適**:
  「TypeScript + Canvas の書き方を学び、自分の作品の土台にしたい」という学習者や開発者にとって最も使いやすく、再利用のハードルが最も低いため、OSSとしての価値が最大化されます。

---

## 2. 他の候補ライセンスとの違い（初心者向け）

### GPL（GNU General Public License）系ライセンスとの比較

| 項目 | MIT License (推奨) | GPL (v3等) |
| --- | --- | --- |
| **ライセンスの性質** | **寛容型 (Permissive)** | **コピーレフト型 (Copyleft)** |
| **二次的著作物の公開義務** | **なし** (クローズドソースOK) | **あり** (改変して公開するならGPL義務) |
| **商業利用のしやすさ** | **極めて容易** (企業の自社製品に組み込める) | **制限あり** (自社コード全体の公開が必要になる可能性) |
| **教育OSSとしての利便性** | 開発者がパーツやロジックを自由に切り取って使える | 派生物もすべてオープンソースにする必要があるため敷居が高い |

---

## 3. 選定決定と採用記録 (Official Adoption)

2026年6月2日、プロジェクトオーナーと開発チームの合意に基づき、本プロジェクトの公式ライセンスとして **MIT License** が正式に採用されました。

### 採用に伴う変更内容：
1. **LICENSE ファイルの追加**: ルートディレクトリに正式な `LICENSE` ファイルを配置。
2. **ライセンス保留ファイルの退避**: 一時的な `LICENSE_PENDING.md` をルートから削除し、歴史的記録として `docs/archive/LICENSE_PENDING.md` に移動（退避）。
3. **ドキュメントの更新**: `README.md` のライセンス記述を MIT License 公開である旨へ変更。

これにより、誰でも安全かつ容易に本ゲームコードを教育、改造、製品の土台としてクリーンに再利用・貢献できるようになりました。
