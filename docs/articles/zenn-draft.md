## なぜ作ったのか？

Webでゲームを作りたいと考えたとき、既存の有名な落ち物パズル（「ぷよぷよ」など）のクローンは著作権や商標、素材ライセンスの壁があります。
「商用や教育用教材としても安全に使える、100%アセットフリーな連鎖パズルゲームを作ろう」というのが本プロジェクトの出発点です。

## 技術スタックと設計

- **コア言語**: TypeScript
- **ビルドツール**: Vite + npm (ランタイムの外部依存ライブラリはゼロ)
- **描画**: HTML5 Canvas (2D)
- **テスト環境**: Vitest (ドロップロジック、盤面消去判定、連鎖計算のロジックを100%テスト)
- **リンター**: ESLint

### 1. アセットフリー（画像・音声ゼロ）
デザインはすべてCanvasを用いた動的なベクター描画で作成しています。ジェムの光沢、消滅時のパーティクル爆発、画面揺れ（Camera Shake）、3Dコンボテキストなど、プレミアムなUXを数式のコードだけで表現しています。

### 2. Web Audio APIによる Procedural Synth
効果音にも一切wavやmp3素材を使っていません。Web Audio APIでオシレータをリアルタイム制御する「 procedural 音源（波形合成）」を搭載しています。連鎖が進むごとに効果音のピッチ（音階）が上がっていく「ド・レ・ミ・ファ・ソ...」の音響的演出もコードのみで実装しています。

### 3. バグのない連鎖アルゴリズムとVitestによるTDD
パズルゲームで最もバグりやすい「ジェムの自由落下」「連結判定（DFS）」「消去」「連鎖開始・ループ処理」のコアモジュールを完全に分離し、Vitestでカバレッジを網羅するテストを記述しています。

## 直近のアップデート (v1.0.2)

最新の **v1.0.2** アップデートでは、GitHub Pages/Live Demoで初めて遊ぶユーザーが迷わないよう、Canvas外に Glassmorphism デザインを採用した「操作方法パネル（フローティングUI）」を追加しました。
PC表示では右下にスッキリと配置し、モバイル画面（900px以下）では下部に綺麗に収まる完全レスポンシブ（390px幅で横スクロールなし）で動作します。

また、既存の快適なキーボード操作（Z/Xで回転、Space/Enter/Pで開始・一時停止）の挙動を全く崩さずにインテグレーションしています。
