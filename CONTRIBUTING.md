# chain-drop-lab 貢献ガイドライン (Contributing Guidelines)

`chain-drop-lab` プロジェクトへの関心とご協力に感謝します！
本プロジェクトは、他社の商標を侵害せず、外部素材（画像・音声ファイル）を一切使わずに純粋な TypeScript + Canvas ロジックだけで構築された、開発者のためのクリーンな教育用ゲームOSSリポジトリです。

共同開発にあたり、プロジェクトの品質とクリーンさを保つための厳格なルールを定義しています。

---

## 1. 開発環境のセットアップ

1. **リポジトリのフォークとクローン**
2. **依存関係のインストール**:
   ```bash
   npm install
   ```
3. **ローカル開発サーバーの起動**:
   ```bash
   npm run dev
   ```
   ブラウザで `http://localhost:5173` を開いて動作を確認します。

---

## 2. 厳格な遵守ルール (最重要)

本プロジェクトに Pull Request (PR) を送る際は、以下のルールを必ず順守してください。

- **第三者の画像・音声・アセット素材を絶対に追加しない**:
  - ゲーム中の演出、ジェム、ボタン、背景などは、すべて Canvas API を用いたプログラムコードでのみ描画します。`.png`, `.jpg`, `.svg` などの画像ファイルをゲーム読み込みアセットとして追加してはいけません。
  - 効果音や音楽も、すべて Web Audio API を用いて `audio.ts` 内でオシレーターから動的生成します。`.mp3`, `.wav`, `.ogg` などの音声ファイルを追加してはいけません。
- **商標権および知的財産権の尊重**:
  - 実在するゲーム名（例: 「ぷよぷよ」「Puyo Puyo」等）や、他社・他ブランドのキャラクターや意匠に酷似した表現を追加しないでください。公開説明は常に「オリジナル落ち物連鎖パズル」に統一します。
- **機密情報 (Secrets) のハードコード禁止**:
  - APIキー、トークン、プライベートIDなどの認証情報は絶対にソースコードに記述しないでください。

---

## 3. 提出前の検証フロー

PRを作成する前に、ローカル環境で以下の統合検証コマンドを実行してください。

```bash
npm run verify
```

このコマンドは内部で以下を順番に実行します。
1. **ESLintによるリントチェック**: 警告やエラーが1件も出ないこと。
2. **Vitestによるユニットテスト**: 全16件のテストが100%パスすること。
3. **Viteによる本番用ビルド**: エラーなくコンパイルが成功すること。

すべてに合格したことを確認した上で、ブランチをプッシュして Pull Request を作成してください。

---

## 4. コミュニティとしての行動規範

私たちは、初心者の開発者が安心して学び、貢献できるオープンで心理的安全性の高いコミュニティを目指しています。
質問やバグ報告、些細な機能提案でも歓迎いたします。お気軽に Issues よりご意見をお寄せください！
